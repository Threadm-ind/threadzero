# Phase 1 Plan: Foundation

## Objective
Establish the narrow v1 foundation for `threadzero` so later phases can add retrieval, prompt packing, execution, and continuation without reworking the core project structure.

## Outcome
A working local CLI with task and checkpoint persistence, config loading, and a minimal lifecycle for creating, viewing, and updating task state.

## Scope
Included:
- repo structure
- TypeScript CLI shell
- config file support
- task schema
- checkpoint schema
- local storage layer
- basic commands for task creation and inspection

Excluded:
- `jcodemunch-mcp` integration
- prompt packing
- executor integration
- checkpoint summarization logic
- benchmark harness
- web UI

## Deliverables
- `package.json`
- `tsconfig.json`
- `src/cli.ts`
- `src/config.ts`
- `src/types.ts`
- `src/storage/db.ts`
- `src/storage/migrations.ts`
- `src/storage/tasks.ts`
- `src/storage/checkpoints.ts`
- `src/task/intake.ts`
- `src/task/checkpoint.ts`
- local data directory or SQLite initialization flow

## Technical Decisions
- language: TypeScript
- runtime: Node.js
- storage: built-in `node:sqlite`
- interface: CLI only
- state model: task-centric, checkpoint-centric
- transcript storage: not primary, keep minimal or none in v1

## Command Surface For Phase 1
Implement these commands:
- `threadzero task new "<goal>"`
- `threadzero task show <task-id>`
- `threadzero task list`
- `threadzero task checkpoint <task-id> --note "<text>"`
- `threadzero checkpoint list <task-id>`

## Verification
- `npm install`
- `npm run build`
- `node dist/cli.js task new "test task"`
- `node dist/cli.js task list`
- `node dist/cli.js task checkpoint <task-id> --note "initial checkpoint"`
- `node dist/cli.js checkpoint list <task-id>`

## Acceptance Criteria
- CLI can create a task successfully
- CLI can list and show tasks
- CLI can create and list checkpoints
- SQLite storage persists across runs
- repo structure matches project intent
- codebase is ready for retrieval integration in Phase 2

## Notes For Execution
Keep everything minimal.
Do not add retrieval, packing, or executor abstractions beyond what Phase 1 strictly needs.
Design for extension, but do not build extension points yet.
